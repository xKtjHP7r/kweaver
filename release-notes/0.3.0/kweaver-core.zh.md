# KWeaver Core 0.3.0 发版说明

本次版本聚焦于业务与技术的深度融合与智能体架构的全面革新。全新的沙箱架构实现了推理与逻辑的职责分离，结合行动驱动可视化与人工干预机制，为构建安全可控、高效可靠的商业级智能体应用提供了全栈式解决方案。

---

## 核心亮点速览

1. **沙箱架构全新升级**：实现了推理（LLM）与逻辑（Code）的职责分离，大幅降低 Token 消耗；整合临时区管理能力，提供会话级数据隔离与中间状态持久化，为智能体构建更安全、稳定、高效的执行环境。
2. **Agent 人工干预**：结合业务知识网络的语义化建模能力，智能体能够根据任务上下文精准召回行动（Action）。针对敏感操作（如审批、删除）引入人工干预机制，确保业务流程"智能而不失控"。
3. **行动驱动能力补齐**：新增可视化的配置与任务管理界面，支持直观管理行动与任务列表，实现从后台逻辑定义到前端可视化管理的跨越。
4. **MCP 标准化集成**：构建 Context Loader MCP Server，基于 MCP 标准提供统一服务接口，仅需配置 URL、Token 和业务知识网络 ID 即可完成集成，支持 Claude Desktop、Cursor、Codex 等主流开发工具。
5. **数据语义能力增强**：VEGA 数据虚拟化引入特征字段，支持为视图字段配置多类型特征（Keyword 精确匹配、Full-Text 全文检索、Vector 向量检索），并提供默认特征机制简化查询操作。

---

## 一、BKN 引擎

**1. 行动驱动前端配置支持**

支持前端配置行动驱动任务，通过可视化界面定义触发规则和执行逻辑，大幅降低使用门槛：

- **事件驱动**：支持基于对象类数据属性的触发条件配置，当满足特定业务规则时主动触发预定义行动，实现事件驱动的智能化业务流程。
- **可视化配置**：采用向导式引导和表单化配置，业务人员无需编写代码即可完成行动驱动的定义。

**2. 任务管理与执行状态查看**

提供任务管理能力，支持查看任务执行状态和结果：

- **任务执行监控**：在任务管理界面可查看行动执行状态，包括总数、成功数、失败数等统计信息。
- **执行记录查看**：支持查看每个行动实例的执行记录，包括执行时间、参数、结果等信息。
- **异步执行支持**：行动驱动采用异步执行机制，任务提交后立即返回，支持批量行动的高效处理。

---

## 二、VEGA 数据虚拟化

**1. 视图支持特征字段**

增强数据语义表达能力，为业务知识网络提供更丰富的数据支撑：

- **多类型特征支持**：支持为视图字段配置多种特征类型：
  - Keyword 特征：适用于精确匹配查询，如 ID、编码等字段。
  - Full-Text 特征：适用于全文检索场景，支持模糊匹配和分词查询。
  - Vector 特征：支持向量化检索，为语义搜索和相似度查询提供基础能力。
- **默认特征机制**：每种特征类型可设置一个默认特征，当查询未指定特征时自动使用，简化查询操作。
- **灵活的特征配置**：支持在视图创建后动态添加或修改特征定义，适应不断变化的业务需求。
- **语义检索增强**：特征字段可被业务知识网络索引和检索，智能体可基于字段语义进行精准的数据查询和推理。

---

## 三、Decision Agent

**1. 与 Context Loader 深度集成**

通过 Decision Agent 与 Context Loader 的深度集成，显著提升行动参数识别准确率：

- **上下文增强**：Context Loader 为 Decision Agent 提供丰富的业务上下文，包括概念定义、实例数据、逻辑属性等，为参数识别提供充足的语义信息。
- **语义推理**：Decision Agent 基于业务知识网络的语义信息进行智能推理，准确识别行动所需参数，减少参数缺失或错误导致的执行失败。

**2. 行动人工干预机制**

针对敏感或高风险的行动操作，提供人工干预机制：

- **人工干预开关**：支持在行动类定义时配置"开启人工干预"选项，敏感操作（如合同审核、数据删除等）在执行前会暂停并等待人工确认。
- **执行确认界面**：人工干预触发时，系统展示行动详情和参数信息，操作人员可选择"执行"或"跳过"。

**3. 临时区改造**

对临时区进行改造，提升中间状态管理的效率与稳定性：

- **临时数据隔离**：为每个智能体会话提供独立的临时数据存储区域，实现会话间的数据隔离，避免数据污染。
- **状态持久化**：支持将中间状态持久化，确保长时间运行的行动不受系统重启或网络波动影响。
- **性能优化**：优化临时区的读写性能，减少状态管理对行动执行效率的影响。

**4. 文件分析能力增强：推理与逻辑职责分离**

全新的沙箱架构为智能体的文件分析场景带来革命性提升：

- **文件直接上传**：文件可直接上传到沙箱环境，无需每轮对话重复下载，显著提升执行效率。
- **Token 大幅节省**：不再将整个文档内容加载到上下文，而是让大模型编写 Python 代码在沙箱中执行分析，对于大型文档可节省 90% 以上的 Token 消耗。
- **智能代码生成**：大模型根据用户问题自动生成数据分析代码，在沙箱中安全执行并返回结果，支持复杂的数据处理和统计分析任务。
- **配置极简化**：仅需配置一个"在沙箱执行代码的工具"，系统即可完成复杂的文件分析任务。

---

## 四、Context Loader

**1. MCP 标准化集成**

构建 Context Loader MCP Server，基于 MCP（Model Context Protocol）标准实现与主流开发工具的无缝集成：

- **标准化配置**：仅需配置 URL、应用账号、用户 Token 和业务知识网络 ID 即可完成集成，完美适配 Claude Desktop、Cursor、Codex 等客户端。
- **工具自动发现**：系统自动加载业务知识网络中定义的所有工具描述信息，智能体可动态发现和调用可用工具，无需额外开发。

**2. Context Loader 与沙箱对接**

实现 Context Loader 与沙箱环境的深度集成，为智能体提供安全可靠的代码执行能力：

- **沙箱函数对接**：通过沙箱环境执行自定义函数，实现代码的安全隔离，防止恶意代码对系统的破坏，确保生产环境的安全性。

---

## 五、Dataflow

**1. Python 运行环境沙箱化**

- **文件上传至沙箱**：支持在节点配置中直接上传文件（如 Excel、CSV），文件自动同步至沙箱环境，方便 Python 代码直接读取与处理。
- **在线代码编辑**：内置 Python 编辑器，支持直接编写数据处理逻辑代码，在沙箱中安全运行，实现灵活的数据清洗与分析。
