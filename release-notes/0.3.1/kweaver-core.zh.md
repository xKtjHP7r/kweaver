# KWeaver Core 0.3.1 发版说明

本次版本聚焦于质量优化与用户体验提升，基于 0.3.0 版本的核心能力，针对用户反馈和生产环境中发现的问题进行全面优化。重点修复 BKN 引擎、VEGA 数据虚拟化、执行工厂等核心模块的稳定性问题，完善决策智能体前端交互体验，确保系统在生产环境中的可靠性与易用性。

---

## 核心亮点速览

1. **数据管理安全性增强**：BKN 引擎新增对象类删除保护机制，通过 `force_delete` 参数控制删除行为，避免因误删导致业务知识网络结构损坏。
2. **系统稳定性全面提升**：全面优化 VEGA 数据虚拟化、Dataflow 数据流、执行工厂等核心模块的异常处理机制，显著提升复杂业务场景下的稳定性与容错能力。
3. **决策智能体交互优化**：优化前端交互体验，临时区支持显隐控制，完善 API 调试功能，统一界面本地化显示。

---

## 一、BKN 引擎

**1. 对象类删除安全控制**

针对对象类删除操作，新增安全控制机制，避免因误删导致业务知识网络结构损坏：

- **引用关系检查**：系统自动检查对象类的引用关系，包括关系类引用和行动类引用。默认模式下仅允许删除未被引用的对象类，有效防止误操作。
- **force_delete 参数**：删除对象类时支持 `force_delete` 查询参数，默认为 `false`。通过 API 接口设置 `force_delete=true` 可强制删除被引用的对象类，适用于特殊场景下的数据清理需求。
- **数据完整性保障**：通过删除控制机制，保障业务知识网络的结构完整性，降低因误删操作导致的系统故障风险。

---

## 二、VEGA 数据虚拟化

**1. 异常数据处理增强**

针对生产环境中的异常数据场景，优化 VEGA 数据视图的异常处理机制：

- **异常捕获完善**：修复数据视图在线上环境对某些异常数据未进行 catch 的问题，防止异常数据导致的服务中断。
- **容错能力提升**：增强数据视图对边界情况和异常输入的处理能力，确保在复杂数据场景下系统的稳定运行。

---

## 三、执行工厂（Execution Factory）

**1. 组合算子版本管理优化**

修复组合算子版本管理相关问题，提升算子执行的可靠性：

- **版本引用一致性**：修复导入组合算子时版本重新生成导致引用该版本的组合算子无法执行的问题，确保版本引用关系的一致性。
- **算子依赖管理**：优化组合算子的依赖关系处理机制，避免版本变更导致的执行失败。

**2. 工具箱访问错误处理**

优化工具箱访问的错误处理机制：

- **状态码规范化**：修复工具箱不存在时返回状态码为 404 的问题，改为更合理的错误提示，提升 API 调用的规范性。
- **错误信息完善**：优化错误提示信息，帮助开发者快速定位问题原因。

---

## 四、Dataflow

**1. 算子循环执行稳定性修复**

修复算子运行中包含循环时的报错问题，提升数据处理流程的可靠性：

- **循环执行支持**：修复算子运行中包含循环时出现的报错问题，确保包含循环逻辑的数据处理流程能够正常执行。
- **流程稳定性提升**：增强 Dataflow 对复杂控制流的处理能力，支持更灵活的数据处理逻辑设计。
- **执行可靠性保障**：优化算子执行引擎，确保循环等复杂逻辑场景下的稳定运行。

---

## 五、Decision Agent

**1. 对话界面临时区优化**

优化 Agent 对话界面的临时区功能，提升界面的灵活性与用户体验：

- **临时区显隐控制**：支持临时区的显示与隐藏切换，用户可根据需要灵活控制界面布局，优化屏幕空间利用。
- **显示异常修复**：修复 Agent 对话界面临时区显示异常的 bug，确保临时区内容的正常展示。

**2. 产品本地化完善**

- **术语统一显示**：页面所有"Decision Agent"统一显示为"决策智能体"，保持产品术语的一致性，降低用户理解成本。

**3. API 调试功能增强**

优化 Agent API 调试功能，解决调试接口的兼容性问题：

- **业务域请求头补全**：修复 Agent API 页面调试接口时缺少 `x-business-domain` 请求头的问题，确保调试请求与实际业务请求的一致性。
- **调试体验提升**：完善调试工具的配置项，为开发者提供更完整的调试能力。

**4. 代码质量优化**

- **组件清理**：移除未使用的 `KNSpaceTree`、`DocTree`、`ContentDataTree` 组件，减少代码冗余，降低维护成本。
