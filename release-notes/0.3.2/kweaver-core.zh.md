# KWeaver Core 0.3.2 发版说明

KWeaver DIP 0.3.2 版本聚焦数据分析能力增强和产品稳定性提升，重点围绕**智能找数能力提升、智能问数体验优化、已有功能 BUG 修复**三个方向进行改进。

---

## 一、智能找数

**1. 找数任务自动规划**
支持将复杂的找数需求自动拆解为可执行步骤，形成完整的任务规划流程。同时通过待办事项管理，全程跟踪任务执行状态，标记已完成/未完成环节，确保关键步骤不遗漏。

**2. 知识增强提升找数准确率**
支持基于业务数据进行知识增强，在找数过程中结合数据资源的元数据信息、样例数据及数据质量探测结果，对召回结果进行校验与筛选，最终输出符合业务需求的数据资源清单，有效提升找数结果的准确率与可靠性。

**3. 基于部门三定职责找数**
支持基于部门三定职责的找数能力，通过查找及匹配数据表的部门信息与信息系统，让数据查找从"关键词匹配"升级为"部门职责驱动匹配"，并在找数结果中输出"负责单位、单位职责、信息系统"等内容，明确数据对应的责任主体和职责依据，确保数据归属清晰。

**4. 意图理解能力优化**
支持精准识别用户问题的核心意图，自动区分找数、追问、反问、意图模糊等场景，匹配对应处理逻辑，让业务人员的需求得到高效、准确的响应。

**5. 自定义搜索排序策略**
支持通过规则库自定义搜索排序策略。当用户问题涉及规则库中设定的关键词时，将自动触发预设策略，实现将指定数据表排在首位或优先推荐。

**6. 基于历史找数结果追问**
支持连续对话下的追问能力。当用户在历史找数结果基础上，通过增加查询条件、缩小查询范围等方式进行追问时，支持基于已有分析结果继续推理，而无需重新开始执行找数任务，使查询逻辑逐步收敛，并有效避免重复计算，提升响应效率与交互体验。

**7. 优化指定限制条件下的找数能力**
支持精准识别用户问题中的限制条件（如"不包含"），在找数流程中严格执行该限制条件，过滤限制条件相关的数据资源，确保找数结果范围符合用户要求。

**8. 找数结果模板化输出**
支持通过提示词定义找数结果输出模板，用户可根据自身业务需求，灵活调整模板内容，使找数结果更加规范且符合业务使用习惯。

**9. 相关问题推荐**
支持回答用户当前提问的同时，自动生成相关延伸问题，帮助用户发现潜在数据查询方向，降低用户表达与输入成本，助力用户全面挖掘数据价值。

**10. 适配数据视图（库表）资源**
找数支持适配数据视图。用户可基于业务场景定位所需视图资源，并通过查看视图详情快速预览相关数据，提升数据查找效率并降低数据理解成本。

**11. 统一找数问数访问入口**
提供统一的智能问数访问入口，用户可通过页面对话框直接发起数据查询和分析任务，默认提供找数能力，同时支持在搜索框下切换不同技能，获取更多数据分析能力。

---

## 二、智能问数

**1. 问数工具体验优化**
提供问数工具箱，打通从问题分析、大模型生成分析代码、沙箱环境执行到结果生成的完整分析流程，同时针对关键环节进行优化，解决沙箱创建失败、知识网络调用异常、Text2SQL 与沙箱工具联通失败的问题，保障分析任务能够稳定运行。

**2. 相关问题推荐**
支持相关问题推荐能力，在回答用户问题的同时，引导用户进一步开展深度分析，帮助用户持续挖掘数据背后的业务价值。

---

## 三、BUG 修复及产品体验优化

**1. Decision Agent 体验优化**
- 支持通过 Docker Compose 方式快速启动 Decision Agent 前端环境。
- 完成 Decision Agent 前端改造，支持不借助 proton 主应用完成本地前后端联调。
- 修复决策智能体开发-我的模板页面下，状态筛选值错误的问题。
- 修复 Agent 对话过程，中断执行后，再次刷新浏览器仍然出现中断面板的问题。
- 修复 Agent 对话过程，取值渲染报错导致页面崩溃的问题。
- Agent 临时区 ID 由动态 id 切换为固定值 `sess-agent-default`，优化临时区沙箱过多导致 pod 资源占用太多的问题。

**2. 业务知识网络问题修复**
- 修复在创建对象类过程中，逻辑属性（Logical Property）无法正常添加的问题，确保对象类创建流程中逻辑属性配置功能的正常使用。

**3. 数据模型 - 指标模型问题修复**
- 修复指标模型中已创建的分组无法执行删除操作的问题，恢复分组管理功能的正常使用。
- 修复复合指标创建无法输入的问题，确保复合指标正常创建及使用。

**4. 数据视图问题修复**
- 修复自定义视图数据管理页面中 UI 元素因层级配置异常导致的样式堆叠显示问题，恢复页面布局的正常显示效果。

**5. 执行工厂问题修复**
- 修复 MCP 详情页面输入参数显示不正确的问题，保障输入的字段、类型、描述等信息的正常展示。
- 修复 MCP 调试接口传参不正确导致调试结果错误的问题，确保工具能够正常调试运行。
